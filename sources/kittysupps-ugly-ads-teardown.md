# KittySupps ugly-ads teardown (method swipe)

Nguồn: /Users/quanghuy/Claude Project/Kittysups. Meta Ad Library, ~1,600 active ads, brand cat-supplement, hero SKU Taurine + Dental ($40-50).

Phương pháp "long-form ugly native advertorial" họ chạy:
- Persona-network: KHÔNG bao giờ chạy từ brand Page. Chạy từ persona xoay vòng. 4 archetype: Peer (confession), Expert (lab demo), Editorial/Institution (study), Cause/Rescue (rant). Tên Page = congruency cue đầu tiên, phải khớp "costume" của hook.
- Decouple copy khỏi persona: 1 script thắng chạy verbatim dưới nhiều persona + re-upload nhiều ID -> sống sót fatigue/ban.
- Hook "ugly" ép "See more": forbidden-truth + chửi bị kiểm duyệt (f*k), self-incrimination ("And I killed him"), false-premise reversal ("TAURINE IS A SCAM"), overheard secret, symptom reframe.
- Body 4,000-5,000 từ, 1 skeleton 15 beat: false-close giải pháp sai -> slippery-slide command -> identity anchor cụ thể -> sensory problem -> failed-solutions montage (brand + giá + timeline) -> escalate tới death + số liệu -> ticking clock con thứ 2 -> mechanism qua DIALOGUE của nhân vật authority (không lecture) -> villain stack -> reason-why numbers -> inoculate vs hàng rẻ Amazon -> objection handling trong story -> transformation theo nhịp (day 6, week 3, vet confirm) -> CTA = rescue + warning, DENY the sale -> P.S. stack.
- Length is the qualifier: 4,000+ từ tự lọc high-intent. Ai tới CTA = pre-sold.
- PDP chỉ là closer. Rủi ro: proof bốc hơi khi tới page (ad hứa COA/lab/named expert, page không có) -> skeptic LTV cao rò rỉ. Fix: advertorial bridge page giữ proof.
