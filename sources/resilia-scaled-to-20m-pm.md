Summary (paraphrased from the source article; ClarFlow's own unsolicited quiz-design proposal for Resilia, ~4.5M visits/mo, ads running across ~12 symptoms and both genders into one product).

**Core mistake diagnosed:** when a brand runs many symptoms/personas/angles, most quiz builders' instinct is to build one giant multi-fork quiz that branches for every possible symptom/persona from day one. The author argues this confuses *personalization* with *complexity* — the more forks you add before you have data, the harder it is to tell which branch is actually driving results. A V1 should be treated as an MVP that concentrates traffic and signal, not a "final, perfect" structure that scatters it.

**The fix used for Resilia:** instead of many branches, build ONE streamlined quiz built to run under ALL the brand's main ad angles at once. The way to make one quiz work across a dozen symptoms is to find the single underlying belief that all the symptom-specific ads secretly share — for Resilia that's a "hidden parasite" mechanism, which becomes the shared explanation behind unrelated-looking symptoms (baggy eyes, puffy face, brain fog, etc). This keeps the quiz broad enough to sit under many different ad angles while staying specific enough to still feel personal.

![[resilia-01.png]]

**Quiz flow should mirror the ad's flow**, not exist independently — the ad already planted a belief; the quiz's job is to deepen it, using the SAME symptom language/moments the winning ads already use (bloating after meals, night sugar cravings, brain fog, low energy). The key technique: replace purely informational yes/no questions ("Do you have sugar cravings?") with **functional/situational questions** that make the person mentally replay the exact moment the problem occurs ("What usually happens at night when cravings hit?"). The claim is that people don't just recall facts, they relive the *emotional state* attached to a remembered situation (frustration, loss of control) — and carrying that state through the quiz means the eventual offer lands on a "warm" emotional register instead of cold logic.

![[resilia-02.jpeg]]

**Mirroring the ad's persuasion pattern, not just its content:** beyond generic frameworks (AIDA/PAS), the author argues the more useful thing to copy is the audience's own *tolerance for repetition* — some audiences convert off one strong proof; this Resilia audience apparently needs the same belief repeated from several different angles before it's accepted (illustrated by a testimonial-style ad about a 101-year-old woman whose whole family shares the same longevity trait, repeating the same underlying claim several times through different family members). The quiz should therefore not state its core belief once — it should seed it early and reinforce it at multiple touchpoints (a reason-why line under a question, an info-slide, a confirming line on the result screen), each phrased slightly differently, so by the time the offer appears it reads as a foregone conclusion rather than a pitch.

![[resilia-03.jpeg]]

**Bottom line / general principle:** for a brand running many ad angles/symptoms/personas into one SKU, the goal of a V1 quiz isn't to cover every variation — it's to find the one belief that connects all the angles, ship a single quiz built on that belief so it can run under every winning ad and gather fast signal, and only add branching complexity once real data justifies it.
